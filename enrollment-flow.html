Enrollment — Complete Data Flow

From user clicking Submit → Servlet → Master API → Response back to browser

1
JS User Clicks Submit — What gets built
The browser collects two things: a JSON string (all demographic text data) and the actual image File objects. These are packed into one FormData object.
📄 "json" text part
applicantId, firstName, lastName
dateOfBirth, gender, city...
group, operation, timestamp
biometrics metadata (filenames)
🖼️ Image file parts
faceFront → File object
faceLeft, faceRight
irisLeft, irisRight
fp_left_thumb, fp_left_index...
fp_right_thumb, fp_right_index...
// JS: handleSubmit()
const formData = new FormData();

// Part 1: JSON string — all text fields
formData.append("json", JSON.stringify(payload));

// Part 2: Actual image files
formData.append("faceFront",    state.face.front.file);   // File object
formData.append("irisLeft",     state.iris.left.file);
formData.append("fp_left_thumb", state.fingerprints.left.thumb.file);
// ... etc

// Send — NO Content-Type header, browser sets it automatically
fetch("enrollment", { method: "POST", body: formData });

2
JAVA Servlet receives the request
Tomcat automatically splits the multipart body into individual parts. Your servlet reads the "json" part as text, and each image part as raw bytes. 
// EnrollmentServlet.doPost()

// ── Read the JSON text ──────────────────────────
String jsonStr = request.getParameter("json");
// → gives you the big JSON string

// ── Parse JSON into Applicant object ────────────
JsonNode root = objectMapper.readTree(jsonStr);
JsonNode demo = root.path("demographic");

Applicant applicant = new Applicant();
applicant.setFirstName(demo.path("firstName").asText());
applicant.setLastName(demo.path("lastName").asText());
// ... all other fields

// ── Read image files ─────────────────────────────
Part facePart = request.getPart("faceFront");
byte[] faceBytes = facePart.getInputStream().readAllBytes();
// faceBytes = raw PNG/JPG image data

// Convert to Base64 to store or send to API
String base64 = Base64.getEncoder().encodeToString(faceBytes);
⚠️ Servlet MUST have @MultipartConfig annotation or getPart() will throw an error!

3
JSON What does the parsed data look like inside Java?
After parsing, you have two Java objects: Applicant (text data) and BiometricData (images as Base64 strings).
Applicant object
applicantId = "uuid..."
firstName = "John"
lastName = "Doe"
dateOfBirth = "1990-05-15"
group = "default"
BiometricData object
faceFront.base64 = "iVBOR..."
faceFront.filename = "face.png"
leftFingerprints["thumb"].base64
leftIris.base64 = "iVBOR..."
// BiometricData is filled like this:
BiometricData biometrics = new BiometricData();

// For each image part → read bytes → Base64 → CaptureSlot
Part part      = request.getPart("faceFront");
byte[] bytes  = part.getInputStream().readAllBytes();
String b64    = Base64.getEncoder().encodeToString(bytes);

biometrics.setFaceFront(new CaptureSlot(b64, "face.png", "0.3MB", "PNG"));

// Fingerprints (loop all 10)
for (String finger : fingers) {
    Part p = request.getPart("fp_left_" + finger);
    if (p != null && p.getSize() > 0) {
        biometrics.getLeftFingerprints().put(finger, readPart(p));
    }
}

4
API Servlet sends to Master API (future step)
You build a JSON body and POST it to the Master ABIS API using Java's HttpClient. The images go as Base64 strings inside the JSON. 
// EnrollmentService.processEnrollment()
// Build the JSON body for the Master API

Map<String, Object> apiBody = new HashMap<>();
apiBody.put("applicantId",  applicant.getApplicantId());
apiBody.put("firstName",    applicant.getFirstName());
apiBody.put("dateOfBirth",  applicant.getDateOfBirth());
apiBody.put("group",        applicant.getGroup());

// Attach biometrics as Base64
Map<String, String> face = new HashMap<>();
face.put("front", biometrics.getFaceFront().getBase64Data());
apiBody.put("face", face);

String jsonBody = objectMapper.writeValueAsString(apiBody);

// Send to Master API via HttpClient
HttpClient client   = HttpClient.newHttpClient();
HttpRequest request = HttpRequest.newBuilder()
    .uri(URI.create("https://master-api.example.com/enroll"))
    .header("Content-Type", "application/json")
    .POST(HttpRequest.BodyPublishers.ofString(jsonBody))
    .build();

HttpResponse<String> apiResp = client.send(request,
    HttpResponse.BodyHandlers.ofString());
✅ Base64 is the standard way to send images inside JSON to REST APIs.
↓
5
JAVA Servlet reads Master API response → sends back to browser
The Master API returns a JSON response. Your servlet reads it, then writes its own JSON response back to the browser. 
// Read what Master API replied
String apiResponseBody = apiResp.body();
int    apiStatus       = apiResp.statusCode();  // 200, 400, 500...

if (apiStatus == 200) {
    // Parse the API response
    JsonNode apiJson   = objectMapper.readTree(apiResponseBody);
    String   refId     = apiJson.path("referenceId").asText();

    // Send success back to browser
    response.setContentType("application/json");
    response.getWriter().write(
        "{" +
        "\"status\":\"success\"," +
        "\"referenceId\":\"" + refId + "\"" +
        "}"
    );
} else {
    // Send error back to browser
    response.setStatus(400);
    response.getWriter().write("{\"status\":\"error\",\"message\":\"API rejected\"}");
}
↓
6
JS Browser receives response → shows result to user
The fetch() .then() handler reads the JSON response from your servlet and shows a success or error message. 
// JS: after fetch()
.then(res => res.json())
.then(data => {
    if (data.status === "success") {
        showToast("Enrolled! Ref ID: " + data.referenceId);
    } else {
        showToast(data.message, true);  // true = error (red)
    }
})
.catch(err => {
    showToast("Network error", true);
});
✅ The browser never refreshes. Everything happens via fetch (AJAX).
